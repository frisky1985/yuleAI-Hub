MCP 协议基础

📅 2026-05-26 📖 阅读约 6 分钟
MCP协议入门JSON-RPC

MCP (Model Context Protocol) 是 AI Agent 之间通信的标准协议。它定义了 Agent 如何发现彼此的能力、如何发送请求、如何交换数据。

为什么需要 MCP？

在没有 MCP 之前，每个 Agent 工具都是独立的——Hermes 有自己的工具调用方式，OpenClaw 有另一套机制。如果要让两个 Agent 协作，需要手动编写胶水代码。

MCP 提供了一个统一的接口层：

  • Agent 通过 MCP 注册自己的工具（能力）
  • 其他 Agent 通过 MCP 发现和调用这些工具
  • 数据和上下文通过 MCP 标准格式交换

核心概念

1. 工具（Tools）

一个工具是一个具名的可调用功能，包含：

  • 名称 — 唯一标识符，如 agent-chat
  • 描述 — 自然语言说明这个工具做什么
  • 参数 Schema — JSON Schema 定义输入参数
  • 实现 — 实际执行的逻辑（Python 函数、Shell 命令等）
// 工具定义示例
{
  "name": "agent-chat",
  "description": "同其他 Agent 进行快速对话",
  "inputSchema": {
    "type": "object",
    "properties": {
      "agent": { "type": "string", "description": "目标 Agent 名称" },
      "message": { "type": "string", "description": "发送的消息" }
    },
    "required": ["agent", "message"]
  }
}

2. 资源（Resources）

资源是 Agent 可以暴露给其他 Agent 的数据，类似 RESTful API 中的资源概念。例如：

  • 文件内容
  • 数据库查询结果
  • 共享上下文片段

3. 提示（Prompts）

预定义的提示模板，帮助其他 Agent 知道如何与你交互。例如一个代码审查 Agent 可能会暴露一个 "review code" 提示模板。

JSON-RPC 消息格式

MCP 使用 JSON-RPC 2.0 作为底层的消息协议。所有消息分为三类：

请求（Request）

{
  "jsonrpc": "2.0",
  "method": "tools/call",
  "params": { ... },
  "id": 1
}

响应（Response）

{
  "jsonrpc": "2.0",
  "result": { ... },
  "id": 1
}

通知（Notification）

{
  "jsonrpc": "2.0",
  "method": "notifications/tools/list_changed"
  // 没有 id——不需要响应
}

MCP Transport

MCP 支持两种传输方式：

方式协议适用场景
stdio标准输入/输出本地 Agent，进程间通信
SSEServer-Sent Events远程 Agent，WebSocket 替代方案

MCP Bridge 使用 stdio 方式启动子进程，通过管道与各 Agent 通信。
