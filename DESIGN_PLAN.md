# yuleAI-Hub 重设计规划文档

## 一、现状分析

### 当前结构
```
首页 (Hero + 卡片聚合) → Know-How(5篇) | Tools(3篇) | Projects(1篇) | 关于
```

### 当前问题
1. **内容层次单一** — 只有 know-how/tools/projects 三个维度，缺少知识库、技能、规则等区分
2. **社区感不足** — 目前更像一个博客，缺少"交流站"的氛围
3. **导航深度不够** — 所有内容平铺，没有知识层级
4. **视觉风格通用** — 腾讯云风格的蓝紫色渐变，辨识度不够

---

## 二、重设计目标

### 核心定位
> **yuleAI-Hub = 个人AI知识家园 + 社区交流站**
>
> 既是作者"我"在学习AI路上的笔记沉淀，也是一个开放给同好交流分享的知识平台。

### 三个关键词
| 关键词 | 含义 | 设计体现 |
|--------|------|----------|
| **🌱 成长型** | 知识在不断积累 | 知识树/时间线/进度感 |
| **🧠 知识型** | 内容是核心价值 | 清晰的信息层级、优秀的阅读体验 |
| **🤝 社区型** | 开放共享交流 | 贡献引导、GitHub联动、参与感 |

## 三、信息架构 (IA) 重构

### 新导航结构

```
yuleAI-Hub
├── 🏠 首页           — 仪表盘 + 知识地图入口
├── 📚 知识库         — 系统性知识 (AI基础/Agent原理/工具原理)
├── 📖 Know-How       — 实践指南 (教程/最佳实践/踩坑记录)
├── ⚡ Skills          — 技能库 (Prompt技巧/Agent配置/工作流)
├── 📋 规则集         — 公约/规范 (命名规范/代码规范/协作规则)
├── 🔧 工具           — 工具配置指南
├── 📁 项目           — 项目文档
└── 💬 关于/社区      — 关于 + 贡献指南
```

### 内容层次说明

| 栏目 | 定位 | 内容特征 | 示例 |
|------|------|----------|------|
| **📚 知识库** | 系统性、理论性 | 长文、结构完整 | 《MCP协议详解》《Agent架构模式》 |
| **📖 Know-How** | 实践性、经验性 | 中篇、步骤清晰 | 《Hermes TUI使用技巧》《DeepSeek优化》 |
| **⚡ Skills** | 原子化、可复用 | 短篇、即插即用 | 《上下文共享模板》《MCP调式技巧》 |
| **📋 规则集** | 约束性、规范性 | 清单、规范文档 | 《代码风格指南》《提交规范》《社区公约》 |

### 标签体系

创建多维度标签分类：
- **领域**: `Agent协同` `MCP` `Prompt` `终端` `API`
- **难度**: `入门` `进阶` `专家`
- **类型**: `教程` `踩坑` `配置` `原理` `案例`
- **工具**: `Hermes` `OpenClaw` `QoderCLI` `Warp`

## 四、视觉设计方案

### 品牌色板

```
主色:      #6C5CE7 (紫) — 代表智慧、创造
辅助色:    #00B894 (绿松) — 代表成长、生机
强调色:    #FDCB6E (暖金) — 代表知识、价值
背景:      暖白 #FAFAFA / 深色 #0D1117
```

选择紫+绿松+暖金的原因：
- 区别于市面上大量蓝/青色的技术站点
- 紫: 智慧和创造力的色彩，适合 AI 主题
- 绿松: 成长和生命力，呼应"知识沉淀"概念
- 暖金: 知识的价值感，也带来温暖和亲切

### 字体策略

```css
--font: "LXGW WenKai", "Noto Sans SC", "PingFang SC", sans-serif;
--mono: "JetBrains Mono", "SF Mono", monospace;
```

- 正文使用霞鹜文楷或思源黑体，提升中文阅读体验
- 代码使用 JetBrains Mono，代码阅读友好
- 标题使用粗重的字重，建立视觉层次

## 五、交互与体验设计

### 知识地图 (Knowledge Map)
- 首页核心组件，用可视化方式展示各个内容领域
- 每个领域显示文章数量、最新更新
- 触摸/悬停展开预览

### 增强搜索
- 站内全文搜索（保持现有 search.json 机制）
- 搜索建议：按标题、描述、标签匹配
- 搜索结果分组显示（知识库/Know-How/Skills）

### 阅读体验优化
- 估计阅读时间
- 文章内锚点导航
- 代码块复制按钮
- 图片点击放大（lightbox）

### 社区互动元素
- GitHub Issue / Discussion 链接
- "编辑此页" 直达 GitHub 编辑
- 文章底部 "分享到 X/Twitter"
- GitHub Star / Fork 徽章

## 六、技术实现方案

### 架构决策 (保持简洁)

```
纯静态 HTML + CSS + JS (无框架)
├── 布局模板: _layouts/default.html
├── 样式: assets/css/style.css (重构)
├── 搜索: search.json (保持)
├── 数据: 各页面直接内联/JSON驱动
└── 托管: GitHub Pages (保持)
```

### 为什么不引入框架/构建工具
- 零依赖，即写即用，降低维护成本
- GitHub Pages 原生支持
- 内容更新只需修改 HTML 或 search.json
- 速度最快，没有构建步骤

## 八、设计原则清单

1. **内容即界面** — 不做过度的装饰设计，内容本身是主角
2. **渐进式呈现** — 先让用户看到最重要的，再逐步展开细节
3. **一致性** — 所有页面共享相同的设计语言和组件
4. **可发现性** — 任何内容最多 3 次点击可达
5. **个人温度** — 保持"一个人的知识花园"的亲切感，不过度工程化
6. **移动优先** — 确保在手机上的阅读体验同样优秀
