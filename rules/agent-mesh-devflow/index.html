📋 Agent Mesh DevFlow 执行规则

双模式协同开发工作流规范 · 2026-05-26

一、执行模式

模式命令适用场景
A. 直接 CLIhermes -z / qodercli -p日常开发、快问快答
B. 后台异步agent-delegate + task-result长任务、批量处理
C. 辅助通信context-write / context-read跨 Agent 信息同步

二、模式选择规则

if 任务预估 < 2 分钟 → 模式 A (直接 CLI)
if 任务预估 > 2 分钟 → 通知用户可选模式 B
if 需要跨 Agent 接力 → 模式 A/B + C (context 传参)

三、7 阶段流水线

需求分析(Hermes) → 排期(Hermes) → 架构(QoderCLI) → 编码(QoderCLI) → 测试用例(Hermes) → 测试执行(Hermes) → 部署(OpenClaw)

四、重试规则

每阶段最多执行 3 次。3 次失败后 OpenClaw 接管并输出异常报告。

五、总结报告

全部完成后输出：需求标题、耗时、执行模式、各阶段状态、产出文件、重试记录。
