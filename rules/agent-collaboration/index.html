📋 Agent 协同规则

📅 2026-05-26 📋 规范 · 适用所有 Agent Mesh 节点
协同规范MCPAgent Mesh

当多个 Agent 通过 MCP Bridge 协同工作时，遵循以下规则可以避免混乱、提高效率。

基本准则

  1. 谁用都行 — Mesh 中任意 Agent 都能通过 Bridge 调用其他 Agent 的工具
  2. 异步优先 — 预估耗时 > 8 秒的任务用 agent-delegate，不阻塞当前会话
  3. 上下文共享 — 重要信息用 context-write 存，不要依赖口头传递
  4. 状态透明 — task-list 随时查看所有进行中的任务，不要重复提交
  5. 失败重试 — 如果 Agent 超时或报错，换一个 Agent 再试

上下文命名规范

context-write 的 key 使用以下格式，避免冲突：

# 格式: 项目_模块_含义
project_db_schema         # 项目的数据库结构
review_code_result        # 代码审查结果
deploy_current_version    # 当前部署版本

任务委派规范

场景方式说明
快问快答（< 8s）agent-chat同步等待，直接返回
耗时任务（> 8s）agent-delegate异步提交，拿到 task_id 后轮询
批量任务多个 delegate并行提交，统一查结果
定时任务delegate + cron委托给具有 cron 能力的 Agent

信息传递规则

  • Agent A 完成分析后，用 context-write 存结果再通知 Agent B
  • Agent B 不直接问 "你刚才做了什么"，而是用 context-read 读取
  • 链式任务中，每个步骤的输入输出都用上下文记录
