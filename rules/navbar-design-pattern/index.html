📋 导航栏设计模式

📅 2026-05-27 📋 模式 · 适用 yuleASR-Configurator
UI导航栏设计模式yuleASR

yuleASR-Configurator 的导航栏采用两行布局方案。本文档记录方案对比和最终选择的设计模式。

方案对比

在确定最终方案前，共对比了 4 种布局方案：

方案描述结论
方案A单行全合一导航❌ 信息过载
方案B侧边栏 + 顶部栏❌ 占用编辑区
方案C两行布局（社区横幅 + 配置器主导航）✅ 选定
方案DTab 式导航❌ 不便于扩展

选定方案：两行布局

┌─────────────────────────────────────┐
│  [社区横幅]  社区名称  |  登录 搜索  │  ← 第一行：社区/通用信息
├─────────────────────────────────────┤
│  [Logo]  功能A  功能B  功能C  [用户] │  ← 第二行：配置器主导航
└─────────────────────────────────────┘

第一行 — 社区横幅

  • 展示社区名称、公告、全局搜索
  • 登录/注册入口
  • 使用 <a href> 跨应用跳转

第二行 — 配置器主导航

  • Logo + 主要功能入口（模块列表、配置、预览等）
  • 用户头像/菜单
  • 内部路由使用 <Link> 组件

链接类型区分规则

场景使用示例
应用内部路由<Link>React Router Link 组件
跨应用/外部跳转<a href>跳转到社区站点、文档站

设计决策记录

  • 为何选两行：社区内容（通用）和配置器功能（业务）天然分层，合并会导致信息密度过高且职责不清
  • 为何不选侧边栏：3 列布局下编辑区宽度已经受限，侧边栏进一步压缩可用空间
  • 链接区分：React Router 的 Link 支持无刷新导航，外部链接用原生 a 保证浏览器默认行为
