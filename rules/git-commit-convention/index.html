📋 Git 提交规范

📅 2026-05-26 📋 规范 · 适用 yuleAI-Hub 仓库
Git规范提交

统一的 Git 提交信息让项目历史清晰可读。本规范参考 Conventional Commits 标准，结合中文项目特点做了适配。

提交格式

<type>(<scope>): <subject>

[optional body]

常用类型

Type说明示例
feat新功能/新内容feat(knowledge): 新增 MCP 协议基础文章
fix修复fix(nav): 修复移动端菜单不关闭的问题
docs文档更新docs: 更新 README 站点结构说明
style样式/设计变更style: 调整卡片 hover 阴影效果
refactor重构refactor(layout): 抽离搜索组件
chore杂项chore: 清理空目录

Scope 范围

Scope说明
knowledge知识库相关
knowhowKnow-How 文章
skill技能库
rule规则集
tool工具配置
project项目文档
nav导航/布局
css样式
search搜索索引

注意事项

  • subject 使用中文，简洁描述做了什么
  • 不要超过 72 个字符
  • 动词开头（新增/修复/更新/删除/重构）
  • 一次提交只做一件事

示例

feat(knowledge): 新增 MCP 协议基础文章

介绍 MCP 的核心概念：Tools、Resources、Prompts，
以及 JSON-RPC 通信流程。

docs: 更新 README 站点结构

feat(skill): 新增上下文共享技巧

fix(nav): 修复移动端搜索框样式异常

chore: 清理 unused 目录
