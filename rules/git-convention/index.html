📋 Git 提交规范

📅 2026-05-27 📋 规范 · 适用 yuleASR-Configurator 项目
Git规范提交yuleASR

统一的 Git 提交规范确保 yuleASR-Configurator 项目的提交历史清晰可读。参考 Conventional Commits 标准，结合项目特点适配。

提交格式

<type>(<scope>): <subject>

[optional body]

常用类型

Type说明示例
feat新功能feat(editor): 新增动态模块实例管理
fix修复fix(navbar): 修复社区链接不跳转的问题
refactor重构refactor(modules): all-modules.ts 改为内联紧凑格式
docs文档更新docs: 更新工作流核心原则文档
style样式变更style: 调整暗色模式对比度
chore杂项chore: 更新依赖版本

提交前检查

  • 必须先运行 pnpm build 确认通过
  • 提交信息使用中文 subject，简洁描述做了什么
  • subject 不超过 72 个字符
  • 动词开头（新增/修复/更新/删除/重构）
  • 一次提交只做一件事，不混用 scope

Scope 范围（yuleASR-Configurator）

Scope说明
editorBSW 模块编辑器相关
modules模块列表/配置相关
navbar导航栏
layout页面布局（3列/全宽）
api后端 API
db数据库相关
docs文档

示例

feat(editor): 新增动态模块实例管理

为每个 BSW 模块创建独立的编辑器实例，
支持动态添加和切换。

refactor(modules): all-modules.ts 改为内联紧凑格式

使用 brace-counting 解析代替正则，
提升解析准确性和可维护性。

style: 调整暗色模式对比度

确保按钮和文本在暗色模式下符合 WCAG AA 标准。
