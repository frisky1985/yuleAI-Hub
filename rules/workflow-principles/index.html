📋 工作流核心原则

📅 2026-05-27 📋 规范 · 适用 yuleASR-Configurator 开发
工作流规范yuleASR

本规则定义了 yuleASR-Configurator 项目开发中必须遵守的工作流核心原则，确保文档先行、质量门禁、沟通高效。

一、文档先行、代码后行

任何功能开发必须遵循 先文档、后代码 的顺序：

  1. 先编写/更新设计文档，明确方案和接口
  2. 文档经评审确认后，再开始编码
  3. 编码过程中如有方案变更，先改文档再改代码

二、UI 改动前必须先做 HTML demo

任何 UI 层改动（新增页面、修改布局、调整组件）必须：

  1. 先制作 HTML 静态 demo
  2. 将 demo 提交给老板预览确认
  3. 老板确认后再进行代码实现

三、实现后提供浏览器预览

每次实现完成后，必须提供可访问的浏览器预览：

  • 本地运行 pnpm dev 确认效果
  • 截图或分享本地 URL 供评审

四、构建通过才提交

代码提交前必须执行构建验证：

pnpm build
  • 构建零错误才允许 git commit
  • 构建警告视同错误处理

五、每次修改意见先改文档

接收评审或反馈意见后的处理流程：

  1. 先更新设计文档，反映修改意见
  2. 再根据更新后的文档修改代码
  3. 最后再次构建验证

执行检查清单

□ 文档已更新
□ UI demo 已确认（若涉及 UI）
□ 实现后浏览器预览通过
□ pnpm build 零错误
□ 修改意见已反映在文档中
