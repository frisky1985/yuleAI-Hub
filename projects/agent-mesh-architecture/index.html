Agent Mesh 架构设计

2026-05-26

架构Agent Mesh设计

Agent Mesh 是一个轻量级的跨 Agent 协同网络，通过 MCP Bridge 实现 OpenClaw、Hermes、QoderCLI 三个 Agent 之间的互相调用和上下文共享。

核心架构

┌─────────────────────────────────────────┐
│              MCP Bridge v2              │
│          (mcp_bridge_server.py)         │
│                                         │
│  ┌──────────┐ ┌──────────┐ ┌────────┐  │
│  │agent-    │ │agent-    │ │context-│  │
│  │chat      │ │delegate  │ │read/   │  │
│  │          │ │          │ │write   │  │
│  └──────────┘ └──────────┘ └────────┘  │
└────────────────┬────────────────────────┘
                 │
    ┌────────────┼────────────┐
    ▼            ▼            ▼
┌───────┐  ┌────────┐  ┌──────────┐
│Hermes │  │QoderCLI│  │ OpenClaw │
└───────┘  └────────┘  └──────────┘

设计原则

  1. MCP 优先 — 所有跨 Agent 通信走 MCP 协议，不引入额外依赖
  2. 异步委派 — 耗时任务异步执行，立即返回 task_id
  3. 共享上下文 — 通过 context-read/write 实现状态共享
  4. 零外部依赖 — 纯 Python 标准库，无需 pip install
  5. 即插即用 — 配置一个 MCP server 即可加入网络

通信流程

  Agent A                    MCP Bridge                  Agent B
    │                            │                          │
    │── agent-delegate(task) ──►│                          │
    │◄── {task_id: "t_abc"} ────│                          │
    │                            │── hermes -z "task" ────►│
    │                            │◄── result ──────────────│
    │── task-result(t_abc) ────►│                          │
    │◄── {status:"completed"} ──│                          │

技术栈

  • Python 3.9+ 标准库
  • MCP stdio 传输协议（JSON-RPC 2.0 + Content-Length 帧）
  • 多线程后台任务执行
  • JSON 文件持久化任务状态和共享上下文
