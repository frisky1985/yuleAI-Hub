🤖 Agent 间直接对话

Agent Mesh v2 的核心升级：Agent 通过文件消息总线直接通信，无需 Orchestrator 中转。

消息流程

Agent A                    Agent B
   │                          │
   ├── send(role, subject)───▶│
   │                          ├── read(inbox)
   │                          ├── reply(msg_id, body)
   │◀─────────────────────────┤
   │                          │

Or (broadcast):
Agent A                          Agent B, C, D...
   │                                │
   ├── broadcast(topic, subject)───▶│  (all subscribers)
   │                                │
   ├── subscribe(role, topic) ─────▶│  (self-register)

支持的通信模式

模式命令场景
点对点send → read / waitArchitect → Planner 发起设计评审
回复replyPlanner → Architect: "设计已评审"
广播broadcastImplementer → Reviewer: "代码待审查"
订阅subscribeReviewer → 订阅 "code/review-request" 主题
线程thread追踪完整对话链
发现discover查找所有可用 Agent

📋 技术债

优先级项影响
P1MCP Bridge 缺少 WebSocket 推送异步任务无法实时通知
P1消息无限增长队列需定期清理（已有 purge 命令）
P2Agent 重连逻辑Bridge 重启后需手动恢复
P2状态无版本控制损坏后无法回滚
P3无消息加密敏感信息明文
P3无自动覆盖率收集无法量化影响
