Codex CLI v0.137+ 只支持 Responses API，而 DeepSeek 只提供 Chat Completions API。 这个本地 Proxy 负责双向协议转换：工具类型重写（shell_command / namespace → function）、角色映射（developer → system）、SSE 流格式转换、以及 reasoning_content 透传。

🚀 自动安装

bash scripts/setup.sh

脚本会执行：

  • 复制 proxy 和启动脚本到 ~/.codex/
  • 写入 config.toml 和 reasoner.config.toml
  • 将 DEEPSEEK_API_KEY 添加到 ~/.zshrc（如果缺）
  • 添加 codex shell 函数到 ~/.zshrc（自动启动 proxy）
  • 安装 macOS LaunchAgent（开机自启）
  • 立即启动 proxy

🧰 手动安装

Step 1: 复制 proxy 文件

cp scripts/deepseek-proxy.py ~/.codex/
cp scripts/start-proxy.sh ~/.codex/
chmod +x ~/.codex/start-proxy.sh

Step 2: 配置 Codex

~/.codex/config.toml：

model = "deepseek-v4-flash"
model_provider = "deepseek"

[model_providers.deepseek]
name = "DeepSeek"
base_url = "http://localhost:8787/v1"
env_key = "DEEPSEEK_API_KEY"
wire_api = "responses"

~/.codex/reasoner.config.toml（用于 --profile reasoner）：

model = "deepseek-v4-pro"
model_provider = "deepseek"

[model_providers.deepseek]
name = "DeepSeek"
base_url = "http://localhost:8787/v1"
env_key = "DEEPSEEK_API_KEY"
wire_api = "responses"

Step 3: 存储 API Key

~/.codex/auth.json:

{
  "auth_mode": "apikey",
  "DEEPSEEK_API_KEY": "your-deepseek-api-key"
}

Step 4: 环境变量

export DEEPSEEK_API_KEY="your-deepseek-api-key"

Step 5: 自动启动 proxy

添加到 ~/.zshrc：

codex() {
    if ! pgrep -f "deepseek-proxy.py" > /dev/null 2>&1; then
        ~/.codex/start-proxy.sh start
    fi
    command codex "$@"
}

Step 6: LaunchAgent（可选）

开机自启 macOS LaunchAgent plist，详见 launchagent.md。

💡 使用

codex                              # 默认: deepseek-v4-flash
codex --profile reasoner           # deepseek-v4-pro
codex -m deepseek-v4-pro           # 显式指定模型

💥 Proxy 核心实现

Proxy 将 Codex 的 Responses API 请求转换为 Chat Completions：

请求转换（Responses → Chat Completions）

  • instructions → system message
  • input 数组 → messages（处理 role 映射 developer→system）
  • Codex 内置工具类型（shell_command, namespace）→ 标准 function 类型
  • max_output_tokens → max_tokens
  • tool_choice.type="function" → 嵌套函数引用

响应转换（Chat Completions → Responses）

  • reasoning_content → Responses reasoning 类型输出项
  • tool_calls → function_call 输出项
  • SSE 流：按 Responses API 的事件序列（response.created → output_item.added → content_part.delta → response.completed）重构
  • usage 数据映射（prompt_tokens → input_tokens, completion_tokens → output_tokens）
