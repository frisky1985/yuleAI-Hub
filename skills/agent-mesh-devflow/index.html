🤖 Agent Mesh DevFlow

三 Agent 协同开发工作流 — 自动化流水线，完成后输出总结报告

核心理念

全程自动化，跨 Agent 自行对话完成。用户只需提需求，等总结报告。

用户提出需求
  ↓
┌─────────────────────────────────────────────────────┐
│                 Agent Mesh DevFlow                   │
│  Hermes ──▶ QoderCLI ──▶ Hermes                     │
│  需求分析 排期  架构设计 编码  测试用例 测试          │
│  自动推进，失败3次由OpenClaw接管                     │
└─────────────────────────────────────────────────────┘
  ↓
用户收到总结报告

7 阶段自动流水线

阶段执行者动作
① 需求分析Hermesagent-chat → 输出需求文档
② 排期Hermesagent-chat → 输出排期文档
③ 架构设计QoderCLIagent-chat → 输出架构文档
④ 编码QoderCLIagent-delegate → 输出代码
⑤ 测试用例Hermesagent-chat → 输出测试文件
⑥ 测试执行Hermesagent-chat → 输出测试报告
⑦ 部署+报告OpenClawgit push + 总结报告

每阶段完成自动进入下一阶段。失败重试3次，仍失败由OpenClaw接管。

失败接管机制

for 阶段 in [需求分析,排期,架构,编码,测试用例,测试执行]:
    尝试 = 0
    while 尝试 < 3:
        try: 执行(); break
        except: 尝试 += 1
    if 尝试 >= 3:
        OpenClaw 接管 → 输出异常报告
        break

触发条件

自动触发: 用户说 "开发需求" / "做一个" / "实现" / "创建项目"
           且需求描述超过 20 字
