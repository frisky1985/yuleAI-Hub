⚡ React 动态实例容器模式 — 多实例 CRUD 完整生命周期管理

📅 2026-05-27 ⏱️ 即用型技能
ReactTypeScript多实例CRUD动态容器

React 组件中父容器可以有多个运行时动态创建的子"实例"，用户可以在运行时创建、重命名、复制、删除、排序这些实例。适用于 AUTOSAR 配置器、资产管理器、或任何层级配置 UI。

适用场景

  • 树形界面需要多实例容器支持（+ / × 按钮）
  • 在配置器中实现动态列表 CRUD
  • 构建类似 Vector DaVinci 的配置器树组件
  • 嵌套动态容器（实例内的子容器也是动态的）

数据模型

模板与实例分离

静态数据定义模板（每个实例有哪些字段）。实例在运行时作为模板的数据副本创建。

// 静态数据（配置 JSON）
interface ConfigContainer {
  id: string; name: string;
  parameters: ConfigParameter[];
  subContainers?: ConfigContainer[];   // 静态子容器
  multiple?: boolean;                   // true = 动态实例容器
  minInstances?: number;                // 默认 0
  // subContainers[0] = 新实例的模板
}

// 运行时状态
interface InstanceData {
  name: string;
  parameterValues: Record<string, any>;  // 每个实例独立的参数值
}

关键规则：当 multiple: true 时，subContainers[0] 是模板，不是预先创建的实例。

实例路径约定

嵌套动态容器的路径格式：

layer:.../container:parent/instance:child_0/container:subContainer/instance:sub_0

⚠️ 路径匹配陷阱：当路径中包含 instance:xxx 段时，不能用完整路径匹配静态树。正确做法是用容器 ID 匹配（而非路径匹配），提取最后一个 container:id 段，在模块的容器树中递归搜索。

实例 CRUD 生命周期

创建（添加）

从模板名称 + 数字后缀生成实例名。新实例深拷贝模板的默认参数值。

删除

从实例列表移除，始终先弹出确认对话框。

重命名

双击名称或右键菜单 → 内联编辑。

复制（含参数继承）

深拷贝源实例的所有参数值，生成新名称。

排序（拖拽）

使用 HTML5 原生拖放 API，用 findIndex 而非 indexOf。

UI 控件

  Instance_0 [⎘] [×]        ← 蓝色复制 + 红色删除
  Instance_1 [⎘] [×]

forwardRef + useImperativeHandle

当树组件内部管理实例状态，但父组件（Editor）需要从右键面板触发实例操作时：

export interface ConfigTreeHandle {
  addInstance: (containerPath: string) => string | undefined
}

export const ConfigTree = forwardRef<ConfigTreeHandle, ConfigTreeProps>(
  function ConfigTree(props, ref) {
    // ... 内部逻辑 ...
    useImperativeHandle(ref, () => ({
      addInstance: (containerPath: string) => {
        // 同步计算实例名，返回给父组件自动选中
        const newName = computeNextName(containerPath)
        addInstance(containerPath)
        return newName
      },
    }), [addInstance, dynamicInstances, config])
  }
)

父组件拿到实例名后立即 setSelectedPath(...) 实现自动选中。

嵌套动态容器

实例的子容器自身也可能是动态的，需要：

  • collectDynamicContainers 递归遍历模板子容器
  • seedNestedDynamics 在父实例被添加时创建嵌套实例条目
  • findTemplateContainer 使用 ID 匹配（而非路径匹配）
  • Self-template 模式：某些 multiple: true 容器无 subContainers，自身即是模板
