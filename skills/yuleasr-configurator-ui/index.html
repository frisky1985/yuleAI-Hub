⚡ AUTOSAR Configurator UI 模式 — React/Vite 配置工具

📅 2026-05-27 ⏱️ 项目技能包
AUTOSARReactTypeScriptZustandyuleASR

构建 AUTOSAR ECU 配置 Web 工具的模式集合（类似 Vector DaVinci Configurator），使用 React + TypeScript + Vite + Zustand 技术栈。涵盖动态多实例容器、ARXML 对齐数据模型、三列编辑器布局、实例状态管理和 localStorage 持久化。

适用场景

  • 构建或扩展 AUTOSAR ECU 配置工具
  • 需要多实例（+ / × / ⎘）的树形配置编辑器
  • 需要离线持久化 + 多配置管理的场景

核心架构

三列布局

col-span-3 | col-span-7      | col-span-2
ConfigTree | Validation+Params | ConfigStatus

数据层级

Module → Container → SubContainer → Parameter
Layer (MCAL / ECUAL / Service / OS)

动态多实例容器

数据模型中用 multiple 标记支持多实例的容器：

interface ConfigContainer {
  id: string; name: string; displayName?: string;
  parameters: ConfigParameter[];
  subContainers?: ConfigContainer[];  // 静态子容器
  multiple?: boolean;                  // true = 动态实例
  minInstances?: number;               // 默认 0
  maxInstances?: number;               // 可选上限
}

Auto-Select 模式

右键面板 [+] 创建实例后，用 forwardRef + useImperativeHandle 返回实例名，自动选中使其参数立即显示：

const handleAddInstance = () => {
  const containerPath = `layer:...`
  const instanceName = configTreeRef.current?.addInstance(containerPath)
  if (instanceName) {
    setSelectedPath(`${containerPath}/instance:${instanceName}`)
  }
}

持久化

单配置自动保存

useConfigStore.subscribe((state) => {
  if (state.isDirty && state.currentConfig) {
    localStorage.setItem('yuleasr_config', JSON.stringify({...}))
  }
})

多配置管理（F-1 模式）

  • yuleasr_config_list — 主列表（ID、名称、时间戳）
  • yuleasr_config_{id} — 每个项目的完整配置数据
  • 支持创建、加载、删除、克隆、导出（.yuleasr.json）、导入

验证引擎

validate()
  ├── validateModuleDependencies()   — module.dependencies 数组检查
  ├── validateOSConfiguration()       — task/ISR/alarm 唯一性
  ├── validateParameterConstraints()  — required/range/enum 检查
  ├── validateCrossModuleReferences() — AUTOSAR 隐式依赖规则
  └── validateRTEConsistency()        — RTE 接口验证

跨模块依赖规则

CanTp → Can, CanNm → Can, CanSM → Can + CanNm,
NvM → Fee + Fls, EcuM → Mcu, Csm → Crypto + CryIf,
Dcm → CanTp, PduR → Can

⚠️ 关键陷阱

  • 命名冲突 — Shadcn/UI 项目中的 primary、secondary 是预定义的 HSL 颜色 token，自定义 key 会被忽略。必须用 app-* 命名空间
  • 嵌套动态容器 — 实例内部的子容器也可能是动态的，需要递归 collectDynamicContainers 和 seedNestedDynamics
  • Self-template 模式 — 某些 multiple: true 容器自身就是模板（无 subContainers），需特殊处理
  • ID 匹配而非路径匹配 — 因路径中包含 instance:xxx 段，查找模板容器必须用 ID 匹配
