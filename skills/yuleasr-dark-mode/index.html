⚡ Dark Mode 实现 — Tailwind React 主题切换方案

📅 2026-05-27 ⏱️ 即用型技能
Dark ModeTailwindReactCSSyuleASR

为 Tailwind React 项目添加暗色模式支持，使用 CSS 自定义属性和非冲突的颜色 token 体系。涵盖 CSS 变量命名、Tailwind 配置、组件级灰色类替换、对比度规则等完整方案。

适用场景

  • 为已有 Tailwind React 应用添加 Dark Mode
  • 修复暗色模式下的颜色对比度问题
  • 搭建主题感知的 CSS 变量系统

CSS 变量系统 — 避免命名冲突

使用 non-conflicting 命名空间（app-* 前缀）避免与 Tailwind 内置调色板冲突：

/* ❌ 错误：与 Tailwind 的 `primary` 颜色冲突 */
--bg-primary: #ffffff;  // Tailwind 把 bg-primary 解析成 hsl(var(--primary))，不是这个变量！
--text-primary: #111827;

/* ✅ 正确：使用 `app-` 前缀避免冲突 */
--app-bg-primary: #ffffff;
--app-text-primary: #111827;
--app-border-primary: #e5e7eb;

Tailwind 配置

在 tailwind.config.js 中定义嵌套颜色：

colors: {
  app: {
    bg: {
      primary: 'var(--app-bg-primary)',
      secondary: 'var(--app-bg-secondary)',
      tertiary: 'var(--app-bg-tertiary)',
    },
    text: {
      primary: 'var(--app-text-primary)',
      secondary: 'var(--app-text-secondary)',
      tertiary: 'var(--app-text-tertiary)',
    },
    border: {
      primary: 'var(--app-border-primary)',
      secondary: 'var(--app-border-secondary)',
    },
  },
}

在 JSX 中使用：bg-app-bg-primary、text-app-text-primary、border-app-border-primary。

步骤

1. 添加 CSS 变量

:root {
  /* 亮色模式 */
  --app-bg-primary: #ffffff;
  --app-bg-secondary: #f9fafb;
  --app-bg-tertiary: #f3f4f6;
  --app-text-primary: #111827;
  --app-text-secondary: #6b7280;
  --app-text-tertiary: #9ca3af;
  --app-border-primary: #e5e7eb;
  --app-border-secondary: #d1d5db;
}

.dark {
  /* 暗色模式 */
  --app-bg-primary: #0f172a;
  --app-bg-secondary: #1e293b;
  --app-bg-tertiary: #334155;
  --app-text-primary: #f1f5f9;
  --app-text-secondary: #f1f5f9;
  --app-text-tertiary: #f1f5f9;
  --app-border-primary: #334155;
  --app-border-secondary: #475569;
}

⚠️ 常见陷阱

固定色背景 + 固定色文字规则

当容器使用固定色背景（如 bg-blue-50），内部所有文字必须使用固定色文字（如 text-blue-700）绝不能使用 text-app-text-primary 等变量 —— 暗色下会变成白色，在浅色背景上不可见。

HSL 主题变量 vs app-* 变量

bg-primary、text-secondary 等类使用 HSL CSS 变量（hsl(var(--primary))），在设置为表面/背景色时要用 app-* 替换。仅用作品牌强调色时保留原样。

双重替换 bug

批量替换脚本跑了两次会导致 bg-app-bg-app-bg-tertiary。跑完后用 grep 检查：

grep -rn 'app-bg-app-bg\|app-text-app-text\|app-border-app-border' src/ --include='*.tsx' | wc -l
