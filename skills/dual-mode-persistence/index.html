⚡ 双模式数据持久化模式 — 离线 local + 在线云端无缝同步

📅 2026-05-27 ⏱️ 即用型技能
离线优先localStorage云端同步ReactHook

一种让功能同时支持离线（localStorage）和在线（API/云端）的模式，在用户登录时实现无缝同步。适用于匿名用户也能使用、登录后跨设备同步的场景。

适用场景

  • 功能需要对匿名用户可用
  • 登录后数据应跨设备同步
  • API 不可用时优雅降级
  • 书签、收藏、阅读列表、最近浏览项、用户偏好

架构

Component Layer (BookmarkButton, PointsDisplay)
         │
    Hook Layer (useBookmarks, usePoints)
         │
Storage Layer (localStorage ◄► API Service)

核心实现

Hook 结构

export function useBookmarks() {
  const { isAuthenticated, token } = useAuth();
  
  // 本地存储（始终可用）
  const [localData, setLocalData] = useLocalStorage('key', []);
  
  // 云端状态（仅认证后）
  const [cloudData, setCloudData] = useState([]);
  const [syncStatus, setSyncStatus] = useState('idle');
  
  // 登录时初始同步
  useEffect(() => {
    if (!isAuthenticated || initialSyncDone.current) return;
    
    const syncData = async () => {
      setSyncStatus('syncing');
      try {
        // 1. 获取云端数据
        const remote = await api.getData();
        
        // 2. 合并本地→云端（本地有但云端没有的项）
        const remoteIds = new Set(remote.map(r => r.id));
        const localToSync = localData.filter(l => !remoteIds.has(l.id));
        
        for (const item of localToSync) {
          await api.addItem(item);
        }
        
        // 3. 刷新并使用云端数据
        const refreshed = await api.getData();
        setCloudData(refreshed);
        setSyncStatus('idle');
      } catch (err) {
        setSyncStatus('error');
      }
    };
    
    syncData();
  }, [isAuthenticated, localData]);
  
  // 返回适当的数据源
  const data = isAuthenticated && cloudData.length > 0 
    ? cloudData : localData;
    
  return { data, syncStatus, ... };
}

已同步绿色云图标 + "已同步到云端"

关键决策

决策原因
本地优先立即更新 localStorage 获取响应式 UI
后台同步API 调用异步执行，不阻塞 UI
登录时合并本地数据优先（对于云端不存在的项）
类型灵活性本地类型适配 API 类型时可使用 as any

两种实现层级

层级示例适用场景
Hook 级（useBookmarks）书签、收藏、阅读列表、最近浏览单功能、用户生成的元数据
Store 级（Zustand）配置、项目、文档、草稿核心应用数据、多组件访问、认证相关

⚠️ 注意事项

  • 使用 isProcessing 状态防止双击导致的竞态条件
  • Token 管理集中在 API 服务中，通过 useEffect 更新
  • 检查 mounted 状态防止 hydration 问题
  • 捕获 API 错误，优雅降级到本地数据
