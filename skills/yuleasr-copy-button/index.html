⚡ React 复制按钮模式 — 永久绿色指示器 + 内容变更自动重置

📅 2026-05-27 ⏱️ 即用型技能
ReactClipboardUXButtonyuleASR

React 组件中"复制到剪贴板"按钮的交互模式。核心设计：复制成功后绿色勾永久显示，直到复制的内容发生变化才自动重置。

适用场景

  • 代码片段、配置文本、生成结果的复制按钮
  • 需要明确告知用户"已复制"状态的交互
  • 表单生成内容、API 返回数据的复制场景

三个状态的状态机

            ┌─────────┐
            │  idle   │ ─── click ──→ ┌─────────┐
            │ [Copy]  │               │ loading │
            └─────────┘               │ spinner │
                 ↑                     └────┬────┘
                 │                   success │
                 │          ┌──────────┐      │
                 │          │  copied  │ ←────┘
                 │          │  ✓ 已复制 │
                 │          └──────────┘
                 │ content changes │
                 └─────────────────┘
  • idle → loading: 用户点击
  • loading → copied: 剪贴板写入成功
  • loading → idle: 写入失败（显示错误，不显示绿色）
  • copied → idle: 复制的内容发生变化（useEffect 依赖）

核心实现

import { useState, useEffect } from 'react'
import { Copy, Check, Loader2 } from 'lucide-react'

function CopyButton({ content, onContentChange }) {
  const [state, setState] = useState('idle')

  // 内容变更自动重置
  useEffect(() => {
    if (state === 'copied') setState('idle')
  }, [onContentChange])

  const handleCopy = async () => {
    setState('loading')
    try {
      await navigator.clipboard.writeText(content)
      setState('copied')
    } catch {
      // Fallback: document.execCommand('copy')
      const textarea = document.createElement('textarea')
      textarea.value = content
      textarea.style.position = 'fixed'
      textarea.style.opacity = '0'
      document.body.appendChild(textarea)
      textarea.select()
      document.execCommand('copy')
      document.body.removeChild(textarea)
      setState('copied')
    }
  }

  return (
    <button onClick={handleCopy}>
      {state === 'loading' ? <Loader2 className="animate-spin" /> :
       state === 'copied' ? <Check className="text-green-500" /> :
       <Copy />}
      {state === 'copied' ? '已复制' : '复制'}
    </button>
  )
}

各状态的 UI 样式

状态按钮样式图标文字
idle默认 outlineCopy复制
loading灰色、禁用Loader2 spin生成中...
copiedbg-green-100 text-green-700 border-green-200Check已复制

重置触发器的选择

内容类型重置触发变量
表单生成内容驱动内容生成的表单状态变量
API 获取内容数据/请求 key（如 config.id）
静态内容无需重置 — 文本永不变化
代码编辑器内容代码文本或哈希值

⚠️ 注意事项

  • 不要用 setTimeout 自动重置 — 绿色勾永久显示，直到内容变化才重置
  • 别忘了 fallback — navigator.clipboard 在非 HTTPS 或部分移动浏览器中不可用，必须有 document.execCommand('copy') 后备方案
  • watch useEffect 依赖 — 使用实际内容来源变量，而不是纯文本字符串。如果每次渲染都重新创建的字符串，effect 会过于频繁触发
