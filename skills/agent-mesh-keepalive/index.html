⚡ Agent Mesh Keepalive

跨 Agent 对话保活与在线维持 · 2026-05-26

确保 OpenClaw、Hermes、QoderCLI 三个 Agent 始终在线、上下文一致，会话不断连。

架构

┌─────────────────────────────────────────┐
│         Agent Mesh Keepalive            │
│  ┌──────────┐  ┌──────────┐  ┌──────┐  │
│  │  健康检查  │  │ 上下文同步 │  │ 保活 │  │
│  └──────────┘  └──────────┘  └──────┘  │
└────────────────┬────────────────────────┘
      ┌────────────┼────────────┐
      ▼            ▼            ▼
  ┌───────┐  ┌────────┐  ┌──────────┐
  │Hermes │  │QoderCLI│  │ OpenClaw │
  └───────┘  └────────┘  └──────────┘

1. 一键检查所有 Agent 状态

agent-ping

或在对话中：

"用 agent-status 检查所有 Agent 是否在线"

2. 保活机制

方式 A：MCP Bridge 保活

在任何 Agent 对话中调用 agent-status 即可。

方式 B：cron 定时保活

# 每 30 分钟自动检查
crontab -l | grep agent-ping

3. 上下文同步

保存当前状态：

"用 context-write，key=任务进度, value=方案A已定稿"

恢复对话：

"用 context-read 查任务进度"

4. 快速恢复断连会话

# Hermes 掉线
hermes -z "继续：$(cat ~/.openclaw/mcp-bridge/workspace/context.json)" --yolo

# QoderCLI 掉线
qodercli -p "先读 ~/.openclaw/mcp-bridge/workspace/context.json 再继续"

# OpenClaw 重启后
直接调用 agent-status 确认网络即可

5. 安装

# 保活脚本
chmod +x ~/.local/bin/agent-ping

# crontab 自动保活
(crontab -l 2>/dev/null; echo "*/30 * * * * ~/.local/bin/agent-ping") | crontab -
三个 Agent 一条心，对话永远不掉线。
