Claude Code + DeepSeek V4

2026-05-30 · 配置ProxyDeepSeek

背景

Claude Code 是 Anthropic 官方的终端 AI 编码助手，但：

  • 在中国大陆 IP 直接访问 Anthropic API 返回 HTTP 403（区域封锁）
  • 香港 IP 也被 Anthropic 封锁
  • Claude Code 二进制硬编码了 Anthropic 专有消息格式，不支持第三方模型

解决方案是配置 DeepSeek 的原生 Anthropic 兼容端点。

核心发现

✅ DeepSeek 原生支持 Anthropic 协议
不需要任何本地代理或格式转换服务。DeepSeek 的 /anthropic/v1/messages 端点直接接受 Anthropic 格式请求，返回 Anthropic 格式响应。

配置步骤

1. 设置环境变量

在 ~/.zshrc（或对应 shell 配置）添加：

# Claude Code + DeepSeek
export ANTHROPIC_BASE_URL=https://api.deepseek.com/anthropic
export ANTHROPIC_API_KEY=sk-你的DeepSeek密钥

然后 source ~/.zshrc 或开新终端。

2. 验证

claude -p 'What model are you running on?'
# → Claude Opus 4.8 (实际跑在 DeepSeek V4 上)

技术细节

端点

POST https://api.deepseek.com/anthropic/v1/messages

鉴权

两种方式都支持：

# x-api-key header（Anthropic 风格）
x-api-key: sk-xxx

# Authorization header（OpenAI 风格）
Authorization: Bearer sk-xxx

支持的参数

  • model — deepseek-chat（自动映射到 deepseek-v4-flash）或 deepseek-reasoner（thinking 模型）
  • max_tokens — 默认 4096，上限按服务端限制
  • stream — SSE streaming（text/event-stream）
  • system — 系统提示词
  • temperature — 0..2
  • top_p
  • stop_sequences
  • tools — Function Calling（name, description, input_schema）
  • tool_choice — none | auto | any | tool

不支持的参数

  • thinking — 忽略
  • top_k — 忽略
  • metadata — 忽略
  • images / documents — 不支持

模型映射

Claude Code 默认发送 claude-opus-4-8 等模型名，这些不会被 DeepSeek 直接接受。但 deepseek-chat 可映射到 deepseek-v4-flash。如果 Claude Code 发的模型名不匹配，可以用兼容层配置拦截。

实测 ANTHROPIC_BASE_URL 指向 DeepSeek，deepseek-chat 返回 deepseek-v4-flash，响应包含完整 Anthropic 格式的内容块（type:"text"、stop_reason、usage）。

踩坑记录

⚠️ 弯路 1：Python 格式翻译代理 最开始写了一个 anthropic_proxy.py（Python stdlib HTTP 服务器），把 Anthropic 格式翻译成 OpenAI 格式调 DeepSeek 再翻译回来。非流式可行，但流式需要自己实现 SSE 转换。这个方案不必要。
⚠️ 弯路 2：litellm 代理 litellm 支持模型映射配置，但需要 Prisma 数据库。即使在 --local 模式下也因 prisma 模块缺失崩溃。用 DATABASE_URL=sqlite:/// 强制本地模式导致 silent exit（exit code 7）。
⚠️ 弯路 3：Clash Verge 代理路由 Anthropic 封锁香港 IP，即使挂了 Hong Kong 节点也返回 403。需要确保 Clash 规则路由 anthropic.com 走美国节点。规则写法和 YAML 引号格式需要特别注意。
✅ 最终方案 DeepSeek 原生支持 Anthropic 协议。直接设 ANTHROPIC_BASE_URL 和 ANTHROPIC_API_KEY 即可。不需要代理服务、不需要额外进程、不需要格式转换。
