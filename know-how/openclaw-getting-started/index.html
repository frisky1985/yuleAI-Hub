2026-05-24

OpenClaw入门

OpenClaw 是一个轻量级的 AI Agent 运行环境。本文介绍基本概念、配置和日常使用。

基本概念

  • Gateway — 消息路由层，管理 session/agent 的通信
  • Agent — AI 模型实例，接收消息并调用工具
  • Session — 一次对话的上下文容器
  • MCP Server — 通过 Model Context Protocol 提供的工具集
  • Skill — 可分享的 Agent 技能包（Markdown 格式）

快速开始

# 查看状态
openclaw --version
openclaw status

# 列出已配置的 MCP 服务器
openclaw mcp list

# 添加一个 MCP 服务器
openclaw mcp set my-server '{
  "command": "python3",
  "args": ["/path/to/server.py"]
}'

配置要点

配置文件位于 ~/.openclaw/openclaw.json：

  • gateway.port — 网关端口（默认 18789）
  • agents.defaults.model — 默认模型配置
  • agents.defaults.workspace — 工作目录

集成 MCP Bridge

将 OpenClaw 接入 Agent Mesh 网络：

openclaw mcp set mcp-bridge '{
  "command": "python3",
  "args": ["~/.openclaw/mcp-bridge/mcp_bridge_server.py"]
}'

配置后即可在对话中调用 Hermes 和 QoderCLI。
