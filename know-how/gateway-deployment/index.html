🔌 OpenClaw Gateway 部署指南

Gateway运维基础架构 · 2026-05-26

OpenClaw Gateway 是连接 Agent、信道、工具和外部客户端的中心网关。启动后解锁子 Agent 调度、心跳保活、Control UI、Cron 定时任务等全部能力。

一键部署

# 1. 安装 launchd 服务（开机自启）
openclaw gateway install

# 2. 启动
openclaw gateway start

# 3. 验证
openclaw gateway status
openclaw status
✅ 健康基线 Runtime: running · Connectivity probe: ok · Capability: read-only
Dashboard: http://127.0.0.1:18789/

部署后解锁的能力

  • 子 Agent 调度 — sessions_spawn 工具可用，Hermes/QoderCLI 可被动态唤醒
  • 心跳周期任务 — 每 30 分钟自动检查邮箱/日历/Agent 状态
  • Cron 定时任务 — 精确到分钟的自动化（替代 crontab）
  • Control UI — 浏览器管理面板 http://127.0.0.1:18789/
  • OpenAI 兼容 API — POST /v1/chat/completions 等端点
  • 信道连接 — 可接入 WhatsApp/Telegram/Discord/Signal
  • MCP Bridge — 自动连接已配置的 MCP 服务器
  • 配置文件热加载 — 修改 openclaw.json 自动生效

Keepalive 整合

启动 Gateway 后，Agent Mesh 保活脚本也新增了 Gateway 检测（每 30 分钟 cron 执行）：

~/.openclaw/mcp-bridge/agent-keepalive.sh

# 状态文件
cat ~/.openclaw/mcp-bridge/workspace/keepalive-state.json
# → {"agents":{"openclaw":"online","hermes":"online",
#               "qodercli":"online","gateway":"online","bridge":"online"}}

自动修复：Gateway 掉线时自动尝试 openclaw gateway restart。

运维命令

openclaw gateway status     # 查看运行状态
openclaw gateway start      # 启动
openclaw gateway stop       # 停止
openclaw gateway restart    # 重启
openclaw gateway install    # 安装 launchd 服务
openclaw logs --follow      # 实时日志
openclaw doctor             # 诊断修复

多网关场景

同一台机器可以运行多个隔离的 Gateway（开发/生产/沙盒）：

OPENCLAW_CONFIG_PATH=~/.openclaw/a.json \
  openclaw gateway --port 19001

OPENCLAW_CONFIG_PATH=~/.openclaw/b.json \
  openclaw gateway --port 19002
  • 每个实例需要唯一的 port、config 路径、state 目录、workspace 根目录
  • 大部分场景一个 Gateway 就够了，多网关只用于隔离需求

远程访问（可选）

需要从外网连接时，推荐 Tailscale：

# openclaw.json 中配置
gateway: {
  tailscale: { mode: "serve", funnel: true }
}

或用 SSH 隧道应急：

ssh -N -L 18789:127.0.0.1:18789 user@host
Gateway 启动 = 从 local embedded 到完整 Agent 平台的关键一步。
