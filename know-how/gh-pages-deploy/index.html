📖 GitHub Pages 部署实战手册

CI/CDGitHub Pages踩坑 · 2026-05-26

yuleAI-Hub 托管在 GitHub Pages 上。这里记录的是经过三次大改、踩了无数坑后验证的「最佳实践」。

正确架构

✅ 已验证的方案 Pages 源设置：「Deploy from a branch: gh-pages」（不是 GitHub Actions 模式）
部署流水线：main 分支开发 → Actions 自动同步到 gh-pages → GitHub 内置 Pages 构建
关键文件：.nojekyll（必须保留，防止 Jekyll 干扰）

工作流文件

.github/workflows/pages-deploy.yml：

name: Deploy Pages

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Deploy to gh-pages
        uses: peaceiris/actions-gh-pages@v4
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: .

部署流程

1 开发
所有代码提交到 main 分支。导航栏、CSS 等全部内联到 HTML（独立文件 GitHub Pages 不兼容）。
2 Push 触发
git push origin main → GitHub Actions 自动运行 → 把 main 内容同步到 gh-pages 分支。
3 GitHub Pages 构建
gh-pages 分支被 push 后，GitHub 内置的 Pages 构建自动开始。
4 CDN 生效
构建完成后上线。CDN 缓存 max-age=600（最长 10 分钟），等缓存过期或手动清除。

验证是否生效

# 检查部署时间戳
curl -sI https://frisky1985.github.io/yuleAI-Hub/ | grep last-modified

# 确认 Actions 成功
# 打开 GitHub → Actions 页面，看 workflow 是否绿色 ✅

导航栏统一标准

⚠️ 常见坑 这是踩了三次的坑，一定记住：
  • auth-section 必须在 nav-inner 内部 — 不要单独放一个 nav-right div 在外面
  • 用 margin-left: auto 右对齐 — 把发布/登录按钮推到导航栏右侧
  • 登录/注册合并为一个「登录」按钮 — 注册在登录页内通过链接跳转
  • 发布按钮始终可见 —「✏️ 发布」放在「登录」左侧，绿色填充区分
  • 子页面 nav 必须与首页统一 — 32 个页面结构一致
  • JS 隐藏登录按钮 — 登录后 loginBtn 隐藏，显示头像下拉菜单

快速排查清单

1. Actions 跑了吗？           → GitHub Actions 页面
2. gh-pages 分支更新了吗？     → git fetch && git log origin/gh-pages
3. 页面时间戳是最新的吗？      → curl -sI | grep last-modified
4. .nojekyll 存在吗？         → gh-pages 分支根目录
5. CDN 等够 10 分钟了吗？     → cache-control: max-age=600
部署不是一次性的，每次都要走这套流程。
