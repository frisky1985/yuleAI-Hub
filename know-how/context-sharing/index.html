跨 Agent 上下文共享方案

2026-05-24

上下文MCP协同

多个 Agent 协同工作时，最关键的痛点是如何共享上下文信息——A Agent 的分析结果，B Agent 需要能读到。

方案：MCP Bridge context-read/write

MCP Bridge 提供了两个工具来实现上下文共享：

写入上下文

# Agent A 写入分析结果
agent-chat(hermes, "用 context-write 把代码审查结果存起来，key=code_review_result")

# 或直接在对话中说
"帮我用 context-write，key=db_schema, value={\"users\":\"id,name,email\"}"

读取上下文

# Agent B 读取上下文
"用 context-read 查一下 code_review_result 是什么"

# 读取全部上下文
"用 context-read"

适用场景

  • 代码审查接力 — Hermes 分析代码 → 存结果 → OpenClaw 读结果做决策
  • 项目状态同步 — 一个 Agent 更新了项目配置，其他 Agent 能读到
  • 多步骤流水线 — 每个步骤的结果存入上下文，下一步骤读取

注意事项

  • 上下文存储在 MCP Bridge 工作区的 context.json 文件中
  • 所有连接到同一个 Bridge 的 Agent 共享同一份上下文
  • 上下文是持久化的，重启 Bridge 后仍然存在
