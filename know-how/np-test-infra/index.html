NP 嵌入式 C 工程 PC 端测试基建

平台: macOS ARM64 · HiSilicon Hi2827A · FreeRTOS · AUTOSAR

背景

NP 平台是 HiSilicon Hi2827A (Cortex-M) 无线 MCU 上的嵌入式 C 项目，约 6400+ 源文件、2.2GB。

不能依赖硬件做测试。所有验证在 PC (macOS ARM64 / Linux x86_64) 上完成，ARM 交叉编译仅用于编译检查。

Mock 策略

手工 mock（不使用 CMock/CMockery）：

Ruby 环境未安装，且 Flatform 已有 Makefile+Unity 手工 mock 先例（PlatOTA/test/）。

  • Std_Types.h: AUTOSAR 类型 (uint8~uint64, Std_ReturnType), 纯 stdint/stdbool, 无 SDK 头
  • Global_Cfg.h: 最小配置集, 所有 GLOBAL_CFG_* 设为 STD_OFF 或默认值
  • pc_test_fix.h: 兼容性修复 — 剥离 __attribute__((section(...))), 映射 MSVC *_s 函数到 C99 等价
  • Os.h: FreeRTOS tick 模拟 — GetCounterValue() 返回测试可控的 Mock_TickCount
  • Usart.h: UART 输出捕获 — 累积 buffer 模式 (append, 不覆盖), 测试可检查发送内容

编译兼容性问题

无需修复源文件，在 mock 层解决：

#问题解决方案
1__attribute__((section(".share_ram"))) 在 Mach-O 上无效pc_test_fix.h 中 #define __attribute__(x)
2MSVC snprintf_s/vsprintf_s (Annex K) 在 macOS 不支持宏映射到 C99 snprintf/vsnprintf
3heap_4.c 用 uint32_t 存指针 → ARM64 截断sed 在构建时替换为 uintptr_t

Sanitizer 配置

通过 CMake 选项全局启用：

  • ASan (AddressSanitizer): 堆/栈缓冲区溢出, use-after-free
  • UBSan (UndefinedBehaviorSanitizer): 整数溢出, 除零, 未定义行为
  • gcov: 行覆盖 + 分支覆盖, lcov 生成 HTML 报告

Valgrind 在 macOS ARM64 不可用, 替代为 Clang 内置 ASan。
