DeepSeek API 性能优化

📅 2026-05-23🏷️ DeepSeek · API · 优化 · 性能

DeepSeek API 调用优化、超时处理和 fallback 配置经验。

响应慢的原因

  • 模型负载 — DeepSeek v4 系列在高峰时段响应可能 >30s
  • 长上下文 — 上下文越长首 Token 耗时越久
  • 工具调用 — Agent 使用工具时增加了一次往返延迟

优化方案

1. 超时配置

# Hermes config.yaml
agent:
  gateway_timeout: 1800       # 30 分钟总超时
  api_max_retries: 3          # 失败重试 3 次

2. Fallback 模型

配置备用模型防止主模型不可用：

# Hermes / OpenClaw 配置
model:
  primary: deepseek/deepseek-v4-flash
  fallbacks:
    - deepseek/deepseek-v4-pro

3. 异步委派（推荐）

通过 MCP Bridge 的异步模式提交任务，不阻塞当前会话：

"用 agent-delegate 把任务交给 hermes，task=分析这份代码"
# 立即返回 task_id，稍后用 task-result 查结果

费用优化

  • DeepSeek v4-flash 性价比高，日常使用首选
  • v4-pro 精度更高但贵，复杂推理才用
  • 启用 Prompt Caching 减少重复计费
