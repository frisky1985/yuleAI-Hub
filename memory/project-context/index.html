🧠 项目核心记忆

📅 2026-05-27 🧠 核心上下文 · yuleASR-Configurator
yuleASR项目上下文架构

本文记录 yuleASR-Configurator 项目的核心记忆信息，供 Agent 在开发时参考。这些信息是框架性的项目认知，不应被遗忘。

项目概览

  • 项目名称：yuleASR-Configurator
  • 领域：AUTOSAR 嵌入式软件配置工具
  • 技术栈：React 19 + Vite 7 + Tailwind 3 + zustand
  • 后端：Fastify + Drizzle + PostgreSQL
  • 架构：monorepo 多包结构
  • BSW 模块数：37 个
  • 当前状态：Sprint 4 融合完成

布局偏好

页面布局采用 3 列布局，比例分配为 3/6/3：

┌──────┬──────────────────────┬──────┐
│  3   │          6           │  3   │
│ 左侧  │      主编辑区        │ 右侧  │
│ 导航  │                      │ 属性  │
└──────┴──────────────────────┴──────┘
  • 左侧（3/12）：模块导航 / 分类列表
  • 中间（6/12）：主编辑区 — Editor 动态实例
  • 右侧（3/12）：属性面板 / 上下文信息

37 个 BSW 模块

项目配置了 37 个 AUTOSAR BSW（基础软件）模块，涵盖以下类别：

  • 系统服务（OS、EcuM、BswM）
  • 存储服务（NvM、Fee、Ea）
  • 通信服务（Can、Lin、Ethernet 协议栈）
  • I/O 服务（Dio、Port、Adc、Pwm 等）
  • 安全和诊断（Dcm、Dem、Fim）
  • 其他基础设施模块

Editor 动态实例管理

  • 每个 BSW 模块拥有独立的编辑器实例
  • 编辑器实例动态创建，按需加载
  • 支持多模块同时编辑，通过 Tab 或面板切换
  • 编辑状态由 zustand store 统一管理

all-modules.ts 解析策略

  • 使用 内联紧凑格式 定义模块配置
  • 解析方式：brace-counting 解析（非正则）
  • 优势：更准确处理嵌套结构，可维护性更好

暗色模式对比度规则

  • 所有 UI 元素在暗色模式下必须符合 WCAG AA 标准
  • 文本对比度 ≥ 4.5:1（正常文本）/ ≥ 3:1（大文本）
  • 按钮/图标等非文本元素对比度 ≥ 3:1

Sprint 4 状态

  • 融合阶段已完成
  • 3 列布局已稳定
  • Editor 动态实例管理已就绪
  • all-modules.ts 已采用 brace-counting 解析
