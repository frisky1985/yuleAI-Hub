基本信息

  • 称呼：老板
  • 角色：yuleASR 产品 / 开发负责人
  • 领域：AUTOSAR 嵌入式软件配置工具

工作风格偏好

  1. 视觉优先：UI 改动必须先看 HTML demo，确认后再进入开发
  2. 文档闭环：所有修改意见必须先反应在文档中，再改代码
  3. 质量敏感：构建必须通过才能提交，不接受"以后再修"
  4. 追求简洁：偏爱紧凑格式（如 all-modules.ts 的内联样式），减少冗余
  5. 注重可访问性：暗色模式对比度必须符合 WCAG AA 标准

沟通建议

  • 提交代码前确保 pnpm build 零错误
  • UI 方案先做静态 demo，附上截图或 URL
  • 代码提交信息写清楚改动内容和原因
  • 文档优先，任何方案的讨论结果先写进文档
