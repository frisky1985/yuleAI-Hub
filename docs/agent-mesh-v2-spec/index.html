Agent 技能清单

每个 Agent 在 ~/.openclaw/mcp-bridge/skills/ 下有独立技能清单：

  • orchestrator.md — 管道编排、状态管理
  • hermes.md — 需求分析、测试
  • qodercli.md — 架构、编码、审查

消息契约

~/.openclaw/mcp-bridge/contracts/:

  • message-v1.json — Agent 间消息格式（must: message_id/from/to/subject/body/timestamp）
  • context-v1.json — 共享上下文格式（must: pipeline_status/current_phase/phase_history）

管道阶段

Phase 0: Init       — 加载状态，检查前置条件
Phase 1: Brainstorm — 需求分析+设计（Hermes 分析 → Architect 架构）
Phase 2: Research   — 代码库探索（Researcher）
Phase 3: Plan       — 任务分解+排期（Planner → Hermes 排期）
Phase 4: Implement  — TDD 编码（Implementer → Tester → Reviewer）↻ ITR
Phase 5: Review     — 最终代码审查（Reviewer）
Phase 6: Report     — 总结报告输出（Orchestrator）

质量门禁

  • 每阶段输出必须有下一个阶段的确认
  • 代码必须通过 TDD 循环（失败测试 → 实现 → 通过 → 审查）
  • 审查与生成必须不同 Agent
  • 阶段 3 次失败 → OpenClaw 接管
