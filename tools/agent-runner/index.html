Agent Runner

📅 2026-05-26
Agent Hermes OpenClaw QoderCLI

Agent Runner 是 Agent Mesh 的任务调度后端，以 Bash 脚本形式运行，负责将异步任务分发给 hermes、openclaw 或 qodercli，执行完成后将结构化结果写入共享工作区，供 MCP Bridge 查询。

下载

从 GitHub 获取最新版本并赋予执行权限：

mkdir -p ~/.openclaw/bin
curl -sL https://raw.githubusercontent.com/frisky1985/yuleAI-Hub/main/tools/agent-runner/agent-runner \
  -o ~/.openclaw/bin/agent-runner
chmod +x ~/.openclaw/bin/agent-runner

安装

将脚本加入 PATH，使 MCP Bridge 能直接调用：

# bash / zsh
echo 'export PATH="$HOME/.openclaw/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc

# 验证
which agent-runner
agent-runner --help 2>/dev/null || echo "agent-runner ready"
MCP Bridge 的 agent-delegate 工具会自动在后台调用 agent-runner，无需手动执行。直接使用 agent-runner 主要用于调试。

参数说明

agent-runner <agent> <task_id> <task_text>
参数类型说明
agentstring目标 Agent 名称，支持 hermes、openclaw、qodercli
task_idstring任务唯一 ID，用于结果文件命名（如 task_20260526_001）
task_textstring传递给 Agent 的任务描述文本，建议用引号包裹

各 Agent 调用方式

Agent实际命令适用场景
hermeshermes -z "<task>" --yolo文件操作、系统任务、自动化
openclawopenclaw session start --prompt "<task>"代码分析、长时任务
qodercliqodercli -p "<task>" --dangerously-skip-permissions代码生成、工程辅助

结果文件

任务执行后，结果写入：

~/.openclaw/mcp-bridge/workspace/tasks/<task_id>.json

文件格式示例：

{
  "task_id": "task_20260526_001",
  "agent": "hermes",
  "status": "completed",
  "output": "任务输出内容（最多 5000 字符）",
  "exit_code": 0
}

status 字段可能的值：

  • running — 任务仍在执行中
  • completed — 任务已完成（exit_code 为 0 表示成功）

使用示例

手动触发任务

# 让 hermes 列出桌面文件
agent-runner hermes task_001 "列出桌面上的所有文件"

# 查看结果
cat ~/.openclaw/mcp-bridge/workspace/tasks/task_001.json

通过 MCP Bridge 委派（推荐）

# 在支持 MCP 的 Agent 中调用 agent-delegate 工具
# MCP Bridge 会自动在后台调用 agent-runner
{
  "tool": "agent-delegate",
  "agent": "hermes",
  "task": "检查 ~/.openclaw 目录结构并输出摘要"
}

故障排查

command not found: agent-runner

脚本不在 PATH 中，检查安装路径：

ls -la ~/.openclaw/bin/agent-runner
echo $PATH | tr ':' '\n' | grep openclaw

若路径未加载，重新执行：

export PATH="$HOME/.openclaw/bin:$PATH"

任务状态一直是 running

Agent 进程可能卡住或崩溃，检查临时输出文件：

# 查看 task_id 对应的输出
cat /tmp/<task_id>.out

# 查看 Agent 进程
ps aux | grep -E "hermes|openclaw|qodercli"

结果文件不存在

确认工作区目录已创建，并检查脚本权限：

mkdir -p ~/.openclaw/mcp-bridge/workspace/tasks
ls -la ~/.openclaw/bin/agent-runner  # 应有 -rwxr-xr-x

exit_code 非 0

查看 output 字段中的错误信息；常见原因是 Agent 未安装或任务文本含特殊字符。建议将任务文本写入文件后通过文件路径传递：

echo "复杂任务描述" > /tmp/task_input.txt
agent-runner hermes task_002 "$(cat /tmp/task_input.txt)"
agent-runner 使用 set -euo pipefail，任何子命令失败都会导致脚本退出。若需容错，可在调用前去掉严格模式，或确保依赖的 Agent 已正确安装。
