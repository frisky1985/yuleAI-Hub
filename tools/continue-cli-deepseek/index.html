Continue CLI + DeepSeek V4 Flash 配置

2026-06-08 · 深度排查 · 踩坑实录

Continue CLIDeepSeekCLI配置踩坑

背景

cn（Continue CLI，@continuedev/cli@1.5.45）安装后默认使用 Continue Hub 下发的 Claude Sonnet 4.6 模型。尽管本地 config.json 配了 DeepSeek，但每次运行 CLI 都报 "invalid x-api-key"。

原始症状

cn "hello" --print → {"error":{"type":"authentication_error","message":"invalid x-api-key"}}

API Key 明明是 DeepSeek 的，却被送到了 Anthropic 的端点。

根因分析

文件优先级

Continue CLI 的配置优先级 和你以为的不一样：

优先级文件说明
🔴 最高~/.continue/config.yamlContinue Hub 拉取的配置文件
🟡 中~/.continue/config.tsTypeScript 钩子，运行时编译
🟢 最低~/.continue/config.json本地 JSON，被前两者覆盖

问题链

  1. auth.json 包含 Hub 登录信息（WorkOS JWT，email: tianhua_ming@126.com）
  2. CLI 启动时用 JWT 向 api.continue.dev 拉取配置
  3. Hub 配置（config.yaml）定义了 anthropic/claude-sonnet-4-6，覆盖本地模型
  4. CLI 用 DeepSeek 的 API Key 去调 Anthropic API → invalid x-api-key

关键发现

即使 config.json 配了 provider: deepseek + model: deepseek-chat，只要 config.yaml 存在（Hub 拉着来的），前者完全被忽略。

修复方案

最终配置

修改 ~/.continue/config.yaml：

~/.continue/config.yaml

name: Local Config
version: 1.0.0
schema: v1
models:
  - uses: deepseek/deepseek-chat
    with:
      DEEPSEEK_API_KEY: sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

config.json 的等价写法（如果 config.yaml 不存在）：

~/.continue/config.json

{
  "models": [
    {
      "title": "DeepSeek V4 Flash",
      "provider": "deepseek",
      "model": "deepseek-chat",
      "apiKey": "sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
    }
  ],
  "allowAnonymousTelemetry": false
}

验证

cn "你是谁？用的什么模型？" --print

应正常返回 DeepSeek 模型输出。

技术细节

Provider 选择

Continue CLI 原生支持 deepseek provider（源码中为 DIt class，继承自 OpenAI provider）。

不要用 provider: openai + apiBase: https://api.deepseek.com/v1，因为 OpenAI provider 会用 x-api-key header，DeepSeek API 要求 Authorization: Bearer。

模型 ID 对比

上下文模型 ID
DeepSeek API（OpenAI 兼容）deepseek-chat
DeepSeek API（Anthropic 兼容）deepseek-v4-flash（OpenClaw 别名）
Continue CLI YAMLdeepseek/deepseek-chat
Continue CLI JSONdeepseek-chat

二进制路径

continue 是 bash 内建关键字。实际 CLI 二进制：

# 安装路径
/opt/homebrew/lib/node_modules/@continuedev/cli/

# 可执行文件
/opt/homebrew/bin/cn   # 指向 @continuedev/cli/dist/cn.js

调试技巧

# 查看日志
tail -f ~/.continue/logs/cn.log

# 查看 Hub 配置
cat ~/.continue/auth.json

# 检查 gpt-5 等 Hub 默认模型是否被使用
grep "modelName" ~/.continue/auth.json

# 加 --print 强制非交互模式（适合 CI 和脚本）
cn "prompt" --print

踩坑路线图

  1. ❌ 改 config.json 的 apiBase → 没用，Hub 覆盖
  2. ❌ 改 config.json 的 provider 为 deepseek → 还是被 Hub 覆盖
  3. ❌ 改 config.ts 的 modifyConfig → Hub 优先级更高
  4. ✅ 改 config.yaml 里的 uses: deepseek/deepseek-chat → 生效
